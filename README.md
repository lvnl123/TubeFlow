## 项目简介

这是一个面向桌面端使用场景的 YouTube 下载工具项目。它的目标不是单纯封装一个命令行，而是把常见下载流程做成清晰、可视、对普通用户更友好的界面化操作。

相比直接手敲 `yt-dlp` 命令，这个项目更强调下面几件事：

- 输入链接后，先快速获取标题、作者、时长等基础信息
- 再进一步拉取可下载的格式列表，避免一开始等待过久
- 默认展示更常用、更容易理解的格式选项
- 把“单独视频”“单独音频”“音视频合并”分开，让操作更直观
- 在下载失败时，尽量自动切换到更可用的下载引擎，而不是直接报错退出
- 把下载过程、合并过程、错误原因都尽量在日志面板中展示出来

如果你希望把这个项目上传到 GitHub 做展示，或者继续把它打磨成更完整的桌面软件，这份代码已经具备不错的基础。

## 项目使用路径

下面这组步骤展示的是这个工具的典型使用路径：

1. 输入 YouTube 链接
2. 快速解析视频信息
3. 查看并筛选可用格式
4. 选择下载模式并开始下载
5. 如有需要自动切换下载引擎
6. 使用 FFmpeg 完成合并并输出结果

## 功能特性

### 1. 两阶段解析体验

项目在解析视频时采用了更偏向桌面体验的两阶段思路：

- 第一阶段先获取基础信息
  - 标题
  - 作者
  - 时长
  - 原始页面链接
- 第二阶段再加载完整格式列表
  - 视频轨格式
  - 音频轨格式
  - 音视频合并候选方案

这样做的好处是用户能更快确认“是不是这条视频”，同时减少等待时的空白感。

### 2. 三种下载模式

#### 单独视频

只下载视频轨道，不包含声音。适合以下情况：

- 后续要自己剪辑
- 想拿纯画面素材
- 准备自行封装或后期处理

#### 单独音频

只下载音频轨道。适合以下情况：

- 保存音乐
- 提取播客、演讲、访谈
- 做字幕、转录或音频分析

#### 音视频

这是最常用的模式。你主要选择视频规格，程序会自动为该视频匹配更合适的音频轨，并通过 `FFmpeg` 完成合并输出。

对于 YouTube 上常见的高清、2K、4K 内容，这种模式尤其重要，因为很多高分辨率资源本身就是音视频分离提供的。

### 3. 格式筛选与格式展示

项目支持对格式列表做更友好的展示与筛选，常见信息包括：

- 格式 ID
- 分辨率
- 帧率
- 容器格式，如 `mp4`、`webm`、`m4a`
- 码率
- 估算文件大小
- 视频/音频编码信息

同时，项目默认倾向于先展示更常用的格式组合，避免用户面对过长、过杂的原始格式表。

### 4. 更稳的下载引擎策略

项目内部支持两条下载链路：

- `yt-dlp.exe` CLI 模式
- Python 内嵌 `yt_dlp` 模式

在实际使用中，不同链路可能会因为以下原因表现不同：

- 本机 `yt-dlp.exe` 版本差异
- 参数兼容性差异
- YouTube 页面策略变化
- SABR 导致的格式缺失
- 某些格式在 CLI 和 Python 模式下返回结果不同

因此项目加入了自动回退策略：

- 自动模式下优先尝试 CLI
- CLI 当前不可用时自动回退到 Python
- 即便手动选择 CLI，在部分失败场景下也会兜底继续尝试 Python

这个策略对提升“能下下来”的成功率很有帮助。

### 5. 高级设置与诊断能力

工具提供了面向真实使用环境的高级设置：

- 下载引擎选择
- 代理设置
- 代理连通性测试
- Cookie 文件导入
- 浏览器 Cookie 读取
- 环境摘要查看

环境摘要中可以看到的重要信息包括：

- 当前引擎模式
- 实际使用的下载引擎
- `yt-dlp` 路径
- `FFmpeg` 路径
- JS runtime 状态
- 当前代理状态
- Cookie 状态

### 6. 更像桌面软件的交互细节

- 下载在后台线程执行，不阻塞主界面
- 支持暂停、继续、取消下载
- 记住最近使用过的链接
- 记住保存目录与设置状态
- 底部日志区可直接查看处理过程
- 失败时能看到更具体的错误信息

## 运行环境

当前项目更适合以下环境：

- Windows
- Python `3.9+`
- 本机可用的 `FFmpeg`

更推荐的环境是：

- Windows 10 / 11
- Python `3.10+`

原因是较新的 `yt-dlp` 在 Python 3.9 环境下可能会出现弃用提示，虽然不一定直接导致功能失效，但整体兼容性和后续维护性会更好。

### 2. 安装 Python 依赖

```powershell
cd E:\learn\youtube_downloader_fresh
python -m pip install -r requirements.txt
```

当前依赖如下：

```txt
yt-dlp>=2025.3.31
PySide6>=6.7
```

### 3. 安装 FFmpeg

这个项目在“音视频”模式下需要使用 `FFmpeg` 合并视频轨与音频轨。

请确保你的系统中满足以下任一条件：

- `ffmpeg.exe` 已加入系统环境变量
- 项目运行时能够检测到本机 FFmpeg 路径

如果没有 FFmpeg，那么：

- 纯视频下载可能仍然可用
- 纯音频下载可能仍然可用
- 但音视频自动合并会受到影响

### 4. 双击脚本安装方式

- `install.bat`
- `start.bat`

但需要注意：这两个脚本当前写死了本机 Python 311 路径。如果你要把仓库发给其他人，建议把脚本改成更通用的调用方式，例如直接使用：

## 三个分页怎么理解

### 单独视频

这个分页只下载视频轨道，不包含音频。

适合：

- 做剪辑素材
- 做封面动图素材
- 后期自己再配音频

### 单独音频

这个分页只下载音频轨道。

适合：

- 保存 BGM
- 提取访谈、播客、演讲
- 做音频转文本前处理

### 音视频

这是推荐大多数人使用的模式。

使用逻辑是：

- 你选择一个想要的视频规格
- 程序根据当前格式列表自动选择更合适的音频
- 最后调用 `FFmpeg` 合并成一个完整文件

这也是应对 YouTube 高清资源“音视频分离”现象的主要方式。

## 下载引擎说明

### 自动模式

推荐默认使用。

行为通常是：

- 优先使用 CLI 模式
- 如果当前 CLI 下载链路不稳定，自动回退到 Python 模式

优点是兼顾速度、兼容性和少折腾。

### CLI 模式

直接调用本机 `yt-dlp.exe`。

优点：

- 更接近命令行原始能力
- 某些环境下启动更直接

缺点：

- 对本机 `yt-dlp.exe` 版本更敏感
- 某些参数支持情况可能不同
- 某些视频在 CLI 下返回的格式集合可能不完整

### Python / helper 模式

使用 Python 内嵌的 `yt_dlp` 链路。

优点：

- 兼容性通常更稳定
- 更适合作为回退方案

适合：

- CLI 模式报错时
- 某些链接在 CLI 下格式不完整时
- 需要兼容旧环境时

  - Edge

## 常见问题

### 1. 提示 `Requested format is not available`

这通常说明：

- 当前选择的格式在本次下载链路中不可用
- CLI 模式返回的格式集与解析阶段不完全一致
- YouTube 的 SABR 策略导致某些格式缺失

当前项目已经针对这类问题做了兼容：

- 先尝试 CLI
- 失败后自动回退到 Python 模式

如果你仍然遇到这个问题，可以继续尝试：

- 重新解析一次格式列表
- 改用“自动”或“Python / helper”模式
- 更新本机 `yt-dlp`
- 检查是否启用了合适的代理或 Cookie

### 2. 提示 `--js-runtimes` 不支持

这通常说明本机 `yt-dlp.exe` 版本不支持该参数。

当前项目已经加入了参数能力探测逻辑：

- 支持时才传参
- 不支持时不再强行传入

所以这个问题在当前版本中通常已经被规避。

### 3. 高清视频下载后没有声音

这是 YouTube 的常见行为，不是项目本身的 bug。

因为很多高清视频本身就是：

- 视频轨单独提供
- 音频轨单独提供

正确方式是使用“音视频”模式，让程序自动选择音频并调用 `FFmpeg` 合并。

### 4. 出现 Python 3.9 的 Deprecated Feature 提示

这是 `yt-dlp` 对旧 Python 版本的兼容提醒。

它的含义通常是：

- 当前还能跑
- 但后续支持会逐渐变差

建议：

- 尽量升级到 Python `3.10+`
- 如果你准备长期维护这个仓库，优先统一到更新版本

### 5. 某些链接无法解析

可能原因包括：

- 网络问题
- 地区限制
- 需要登录态
- YouTube 页面策略变化
- 本机 `yt-dlp` 版本过旧
- 代理不可用
- Cookie 不完整

建议排查顺序：

1. 先换一个公开视频试试
2. 再检查代理是否可用
3. 再检查 Cookie 是否正确
4. 最后检查 `yt-dlp` 版本是否过旧
